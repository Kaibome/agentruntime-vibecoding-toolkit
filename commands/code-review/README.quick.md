# /code-review 团队速查版

## 什么时候用

- Python 代码改完后
- 提交 MR / PR 前
- 想快速做一次质量兜底

## 一条命令

```text
/code-review
```

## 它会做什么

- 自动读取当前 `git diff`
- 聚焦变更中的 `.py` 文件
- 调用 `python-reviewer` 做审查
- 输出问题和修复建议

## 看结果时先盯这 4 类

- 安全问题（注入、路径穿越、反序列化等）
- 错误处理（裸 except、吞异常、资源未正确关闭）
- 类型注解（公共函数类型缺失或不准确）
- 性能与风格（低效写法、非 Pythonic 写法）

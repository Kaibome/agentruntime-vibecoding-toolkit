# /code-review 命令使用说明

`/code-review` 是一个用于当前代码变更审查的命令，主要面向 Python 代码质量检查。该命令会调用 `python-reviewer` agent，重点覆盖安全、风格、类型注解与性能问题。

## 命令定位

- 针对“当前变更”执行代码审查，而不是全量历史代码。
- 重点关注 Python 文件改动。
- 目标是提前发现高风险问题，降低提交后返工成本。

## 适用场景

- 完成代码修改后，准备自检。
- 提交 MR / PR 前，做一次质量兜底。
- 需要“第二双眼睛”快速检查时。

## 使用方式

在命令输入区执行：

```text
/code-review
```

## 执行行为

触发后会自动：

1. 运行 `git diff` 获取当前变更。
2. 识别并聚焦变更中的 Python 文件。
3. 调用 `python-reviewer` 对相关文件进行审查。
4. 输出问题清单与修复建议。

## 审查重点

- **安全**：如 SQL 注入、命令注入、路径穿越、不安全反序列化等。
- **代码风格**：命名规范、结构可读性、Pythonic 写法。
- **类型注解**：公共接口类型完整性与准确性。
- **性能**：常见低效写法与可优化点。

## 推荐团队实践

- 将 `/code-review` 作为“提交前固定动作”。
- 对高优先级问题（安全、异常处理）优先修复。
- 评审意见保留到 MR 讨论中，便于追踪与复盘。

## 常见问题

- **为什么没有审查到某些文件？**  
  该命令基于当前 `git diff` 变更范围，未变更文件通常不会进入本次审查。

- **是否只适用于 Python？**  
  当前命令说明与审查代理定位均为 Python 代码审查。
